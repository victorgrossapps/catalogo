# Protocolo de pruebas con PDFs reales en tablet Android

## Objetivo

Validar dos o tres catálogos reales antes de cerrar la decisión del visor PDF y el límite operativo de peso del MVP.

## Catálogos a probar

| Catálogo | Peso | Páginas | Resultado |
|---|---:|---:|---|
| Catálogo real 1 | Pendiente | Pendiente | Pendiente |
| Catálogo real 2 | Pendiente | Pendiente | Pendiente |
| Catálogo real 3 | Opcional | Opcional | Pendiente |

## Pruebas obligatorias

- Tiempo de descarga con Wi-Fi real.
- Tiempo de apertura desde almacenamiento local.
- Fluidez del cambio de página.
- Zoom y lectura de textos pequeños.
- Comportamiento en orientación horizontal.
- Comportamiento en orientación vertical.
- Uso sin internet después de descargar.
- Recuperación ante descarga interrumpida.
- Validación de checksum y descarte de archivo corrupto.
- Consumo de almacenamiento en tablet.

## Criterio de aceptación

El visor PDF del MVP se considera válido si abre el catálogo real con fluidez aceptable, permite zoom cómodo, no bloquea la interfaz y mantiene funcionamiento offline estable.

La prueba física requiere tablets Android reales y PDFs comerciales definitivos o representativos. Este protocolo queda listo para ejecución en campo.
